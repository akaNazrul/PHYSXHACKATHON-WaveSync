Title & Problem
Product Title WaveSync — Real-Time Wave Interference Simulator

The Problem It Solves

Wave interference is one of the most fundamental phenomena in physics — it governs acoustics, optics, quantum mechanics, wireless signals, and medical imaging. Yet for most students and learners, it exists only as a static equation in a textbook or a grainy classroom diagram.

There is no fast, frictionless tool that lets a user feel what happens when two waves collide — to instantly see the nodes, antinodes, and pressure fields shift as they change frequency, amplitude, or source position. Existing simulators are either locked behind clunky Java applets, require downloads, or are too academic to engage with quickly.

Core Insight WaveSync turns a two-variable physics equation into a tactile, real-time visual experience. Zero friction. No login. Open the URL and the physics is already alive on screen.

Real-World Angle

Noise-cancelling headphones, 5G beamforming antenna arrays, ultrasound medical imaging, Wi-Fi dead zones in your home — all governed by wave interference. WaveSync is the explainer tool these conversations have always needed.

Product Overview

WaveSync is a fully client-side, browser-based physics simulation. The user opens the app, sees a live 2D interference heatmap generated by two configurable wave sources, and can manipulate every variable in real time. No backend, no database, no authentication — just physics rendered at 60fps.

Key Features & MVP

MVP Feature Set (ships within 4 hours)

F-01
Live interference heatmap

Canvas renders superposition amplitude across a 2D grid. Colour ramp: deep blue (destructive) through black to bright amber (constructive).

F-02
Dual wave source control

Per-source sliders for frequency (0.5–5 Hz), amplitude (0.1–2), and phase offset (0–360°).

F-03
Drag-to-reposition sources

Click and drag either source dot on the canvas to reposition. Pattern updates immediately.

F-04
Equation info panel

Toggleable overlay showing the superposition formula with live variable substitution as sliders move.

F-05
Physics presets

Three one-click presets: Young's double-slit, noise cancellation, Wi-Fi antenna array. Instantly loads meaningful parameter sets.

F-06
PNG snapshot export

Single button downloads current canvas state as a timestamped PNG for report and video use.

Post-MVP (if time permits)

Stretch goals Animated time evolution of the wave field (toggle play/pause). Third wave source. Shareable URL encoding current parameters. Mobile touch support.

External APIs

None. Zero runtime API calls. All data is computed locally in the browser. This is a deliberate design choice — it makes the app fully offline-capable and eliminates network failure as a demo risk on hackathon day.

Icons

Lucide icon set loaded via CDN (cdn.jsdelivr.net/npm/lucide@latest). Used for control panel actions: play, pause, download, info, reset. No custom SVG assets required — eliminates design bottleneck.

Layout

Two-column layout on desktop. Left column (70% width): the full-height Canvas rendering the interference heatmap. Right column (30% width): control panel with per-source sliders, preset buttons, and equation display. On narrower viewports, controls collapse below the canvas.

Typography

DM Mono throughout — reinforces the "scientific instrument" feel. Equation display uses a slightly larger monospace weight to feel like a readout. No serif, no sans-serif system fonts. Every label should feel like it was printed on a CRT display.

Animation

The heatmap itself is the primary animation — continuous colour transitions as wave patterns update. Source dots pulse subtly with a CSS keyframe ring to indicate they are draggable. Slider value changes trigger a 100ms canvas fade-in (prevents jarring cuts). No decorative animation beyond these functional moments.

Icons (Lucide Animated)

Use Lucide icons for: Download (export), Info (toggle equation panel), RotateCcw (reset to defaults), Play/Pause (if time animation is added). Icon size 16px, stroke 1.5px, colour inherits from parent — adapts cleanly to the dark panel background.

Backend & Database
Architecture decision WaveSync has no backend and no database. This is intentional and is a competitive advantage for this project — it eliminates all infrastructure risk, simplifies the GitHub submission, and makes the demo 100% reliable regardless of network conditions.

State Management

All application state lives in a single JavaScript object in memory, updated by the control bindings and read by the renderer on each animation frame. No localStorage, no IndexedDB — the app resets to defaults on page load, which is the correct behaviour for a simulation tool.

Authentication
No authentication required WaveSync is a public, open-access tool. There are no user accounts, no sessions, no API keys, and no gated features. The decision is deliberate — maximum accessibility is a design principle, not an oversight. Any person with a browser URL can use the full product.

Scalability

WaveSync scales horizontally to unlimited concurrent users at zero cost — being a static site, each user's browser runs their own simulation independently. There is no server to scale.

Computation Scalability

The current physics engine is O(n²) where n is the canvas grid dimension. At 400×400 (160,000 cells), each frame takes approximately 8–12ms on a modern laptop — well within the 16ms budget. If higher resolution is desired, the computation can be offloaded to a Web Worker to prevent blocking the main thread, or partially parallelised using SharedArrayBuffer. These are clean post-submission improvements that require no architecture changes.

Feature Scalability

Adding a third wave source, 3D visualisation via WebGL, or a time-animation playback are all additive features that do not require restructuring the current module boundaries. The state object simply grows by one more source entry.
